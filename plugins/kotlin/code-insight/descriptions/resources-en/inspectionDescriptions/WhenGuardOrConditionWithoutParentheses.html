Reports guard conditions with the || operator without clarifying parentheses.

The || operator can be confusing when used in guard conditions without parentheses. In a branch with a guarded condition a if b || c the operations have the following priority: a if (b || c). In a branch with a regular condition a && b || c the precedence is different: (a && b) || c. To avoid confusion, clarifying parentheses are recommended.

Example: 


fun example(param: Any, flag: Boolean) {
    when (param) {
        is Int if param < 0 || flag -> println("foo")
        else -> println("bar")
    }
}

After the quick-fix is applied: 


fun example(param: Any, flag: Boolean) {
    when (param) {
        is Int if (param < 0 || flag) -> println("foo")
        else -> println("bar")
    }
}
